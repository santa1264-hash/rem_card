# Версионность РЕМКАРТА

Версия хранится в файле `VERSION` в формате `MAJOR.MINOR.PATCH`.

- `PATCH`: небольшие исправления и точечные изменения, например `1.0.1` -> `1.0.2`.
- `MINOR`: крупное обновление или заметная новая функция, например `1.0.2` -> `1.1.0`.
- `MAJOR`: очень большое обновление или несовместимое изменение, например `1.1.0` -> `2.0.0`.

Перед коммитом изменения версии делаются одной командой:

```powershell
python scripts\bump_version.py patch "Исправлена печать карты"
```

Для крупного обновления:

```powershell
python scripts\bump_version.py minor "Добавлен новый раздел назначений"
```

Для очень большого обновления:

```powershell
python scripts\bump_version.py major "Переход на новую структуру данных"
```

Скрипт обновляет:

- `VERSION` — текущий номер версии.
- `CHANGELOG.md` — полный журнал версий и изменений.
- `app/release_info.json` — текущая версия и список изменений в машинно-читаемом виде.

Заголовок окна РЕМКАРТА берет номер версии из `VERSION`. PyInstaller-сборка добавляет этот файл через `RemCard.spec`, поэтому в exe отображается та же версия, что была в dev-версии на момент сборки.
