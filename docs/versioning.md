# Версионность РЕМКАРТА

Версия хранится в файле `VERSION` в формате `MAJOR.MINOR.PATCH`.

- `PATCH`: небольшие исправления и точечные изменения, например `1.0.1` -> `1.0.2`.
- `MINOR`: крупное обновление или заметная новая функция, например `1.0.2` -> `1.1.0`.
- `MAJOR`: очень большое обновление или несовместимое изменение, например `1.1.0` -> `2.0.0`.

Для релизной сборки используйте одну команду:

```powershell
python scripts\build_release.py
```

Она автоматически:

- берет git-коммиты после прошлого изменения `VERSION`;
- добавляет их в `CHANGELOG.md`;
- поднимает версию;
- обновляет `app/release_info.json`;
- запускает `PyInstaller`;
- создает release-коммит вида `Релиз 1.2.3`.

Обычные рабочие коммиты теперь делаются без ручного изменения версии. Пишите сообщения рабочих коммитов на русском языке: релизный скрипт видит эти коммиты и формирует из них changelog.

По умолчанию уровень версии определяется автоматически по сообщениям коммитов. Для явного уровня можно указать:

```powershell
python scripts\build_release.py patch
python scripts\build_release.py minor
python scripts\build_release.py major
```

Если нужно сразу отправить release-коммит в GitHub:

```powershell
python scripts\build_release.py --push
```

Если нужно добавить ручной пункт в changelog:

```powershell
python scripts\build_release.py --change "Исправлена печать карты"
```

`scripts\bump_version.py` остается низкоуровневым вспомогательным скриптом для ручного исправления версии. Для обычной сборки используйте `scripts\build_release.py`, а не прямой `pyinstaller RemCard.spec`, потому что прямой `pyinstaller` не поднимает версию и может создать пакет обновления со старым номером версии.

Заголовок окна РЕМКАРТА берет номер версии из `VERSION`. PyInstaller-сборка добавляет этот файл через `RemCard.spec`, поэтому в exe отображается та же версия, что была в dev-версии на момент сборки.
