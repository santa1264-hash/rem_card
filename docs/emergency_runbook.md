# Аварийная инструкция

## БД недоступна

1. Не запускать recovery вручную.
2. Проверить доступ к сетевой папке.
3. Проверить права на `Baza_rao3_jurnal/archiv`.
4. Закрыть лишние экземпляры приложения.
5. Повторить запуск.

## БД занята

`locked`, `busy`, `unavailable` не равны corruption.

1. Дождаться завершения операции на другом ПК.
2. Проверить, не открыт ли второй экземпляр на той же роли.
3. Не восстанавливать БД из backup только из-за lock.

## Нужна новая версия

1. Закрыть текущий клиент.
2. Обновить приложение.
3. Запустить снова.
4. Если второй ПК еще старый, обновить и его до открытия БД.

## Recovery warning

Auto-recovery запрещен, если активен второй клиент. В этом случае восстановление выполняется только вручную ответственным после закрытия всех клиентов и проверки backup через `restore_drill.py`.

## Где искать логи

- локальные метрики: локальная папка логов клиента;
- backup/recovery audit: `Baza_rao3_jurnal/logs`;
- backup validation reports: `Baza_rao3_jurnal/backup_health/reports`;
- invalid backups: `Baza_rao3_jurnal/backup_health/invalid_backups`.
